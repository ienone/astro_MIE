## 发布后检查

部署完成后按顺序访问：

1. `/`：首页是否显示背景、作者和最近文章。
2. `/posts/`：文章列表是否分页、排序和展示摘要。
3. `/tags/`、`/categories/`、`/series/`：术语索引是否正常生成。
4. `/rss.xml`：RSS 是否包含已发布文章。
5. `/robots.txt`：是否包含正确 sitemap 地址。
6. `/sitemap-index.xml`：搜索引擎 sitemap 是否存在。
7. 文章详情页：hero、TOC、分享、作者、相关文章和上下篇导航是否正常。

## SEO 检查

每次正式发布后建议抽查任意一篇文章的 HTML head：

- `<title>` 是否包含文章标题和站点名；
- description 是否来自 `seo.description`、`description` 或摘要兜底；
- canonical 是否指向正式域名；
- OG 图片是否可访问；
- verification meta 是否按配置输出；
- analytics 脚本是否只在生产构建中出现。

媒体检查重点：

- featured 图片在文章卡片和文章页封面中正常显示；
- background 图片在 hero 背景中正常显示；
- `media.alt` 不为空；
- 图片路径使用站点可公开访问的路径；
- 默认图片可作为缺图兜底。

## 功能验收矩阵

| 功能 | 入口 | 验收标准 |
| --- | --- | --- |
| 搜索 | 顶部搜索按钮 | 能搜索到已发布文章标题和正文 |
| 代码复制 | 代码块 | 按钮可见且不会破坏代码布局 |
| 图片灯箱 | 文章图片 | 点击图片可打开预览 |
| TOC | 文章页 | 多个二级标题时显示目录 |
| 分享链接 | 文章页 | Email、Telegram、Twitter、Reddit 等链接可打开 |
| RSS | `/rss.xml` | 包含正确标题、摘要、日期和链接 |
| Sitemap | `/sitemap-index.xml` | 构建后存在且地址可访问 |
| Robots | `/robots.txt` | 包含 Allow/Disallow 和 sitemap 信息 |

## 回滚建议

如果发布后出现问题，优先执行以下步骤：

1. 回到上一条已知可用 commit；
2. 本地运行 `npm run build` 确认问题是否可复现；
3. 检查最近变更是否涉及 `blog.config.mjs`、内容 front matter 或 Astro 路由；
4. 修复后重新提交并等待 CI 部署。

这份清单应随主题能力一起维护，确保它始终覆盖真实部署流程，而不是只作为示例文章存在。
