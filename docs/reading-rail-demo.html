Reading rail demo

Fused progress and scrollbar

A standalone prototype for the article page. The right rail combines total scroll range, read progress, a current-position endpoint, section markers, and a contextual heading label.

Desktop interaction Rail Hover, drag, and jump between sections.

Article body

Overview

The rail treats the page as a vertical reading map. A quiet base line represents the total article, a brighter line from the top represents progress, and the thicker pill represents the current read position. The endpoint pill can be dragged without smooth-scroll delay.

The interaction is intentionally calm while reading. It becomes more expressive only when the user scrolls, hovers near the rail, drags the endpoint, or points at a section marker.

State model

The visual states are idle, scrolling, hover, dragging, and settling. Idle stays visible but low contrast. Hover raises contrast and makes the active label available. Dragging expands the endpoint and pins the label to the current section.

The important distinction is geometric: every marker and the current progress endpoint sits on the same rail axis. This keeps the control readable as a single fused component.

Progress layer

The read layer is drawn over the total rail from top to bottom. It is slightly thicker than the base line, and the endpoint pill sits across its lower edge. This keeps progress readable without overpowering the article.

The color uses the Material primary tone. On a real article page, this can inherit the post's dynamic color seed, which would tie the rail to the cover image or section theme.

Unread layer

The unread part remains as a muted outline line. It does not need a separate strong color. The contrast difference between the base line and the read layer is enough to show progress.

Progress endpoint

The horizontal pill marks the lower edge of the read range. It is centered on the same axis as the rail and section dots, so the entire control reads as one continuous vertical instrument.

In dragging state, the endpoint expands into a larger Material pill with higher elevation. Its movement is direct and synchronous. Only width, shadow, and color are animated.

Dragging behavior

During drag, global smooth scrolling is temporarily disabled. This prevents queued smooth scroll updates from lagging behind the pointer. On release, smooth scrolling returns for marker clicks and normal anchor navigation.

Section markers

Headings become nodes on the rail. Larger dots indicate higher-level sections, and smaller dots indicate nested sections. Read markers pick up the primary color; unread markers remain quiet.

A single floating tooltip is used instead of placing one label inside every marker. The single-label approach is easier to clamp near the viewport edges and avoids overlapping labels in dense articles.

Active label

The label is intentionally transient. It stays open while hovering a section dot or the label itself, and when the progress endpoint rests on a section dot. Passing a section while scrolling only triggers a short, smooth reveal before it fades back out.

Mobile behavior

On narrow screens, the rail is hidden and the top progress strip returns. This is a practical compromise: side labels have little room on mobile, and a fixed right-side control is easy to hit accidentally during touch scrolling.

The same progress variable can drive both desktop rail and mobile top strip, so the implementation does not need two separate progress systems.

Reduced motion

When reduced motion is requested, the rail should still be fully functional. It should remove scale and slide transitions while preserving opacity and direct drag behavior.

This keeps the component accessible without giving up the Material interaction model for users who want motion.

Implementation notes

The prototype is intentionally self-contained. The production version can reuse the same metrics logic in the Astro scroll component and replace the existing top progress bar on desktop article pages.

The next useful iteration would be testing it against real post titles, long Chinese headings, and dense heading structures to tune tooltip width, marker density, and color contrast.

What to inspect Scroll the article, hover the rail, drag the pill, and hover section dots. The top progress bar is intentionally only used below mobile width.
